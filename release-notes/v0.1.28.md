# AgentDock 0.1.28

## 中文

本版本汇总 `v0.1.25` 之后的改进：修复托管客户端统计为空、Grok 项目目录加载，以及火山 AgentPlan 开启 1M 模型标记后测试返回 404 的问题。

### 新增功能

- 使用统计同时读取客户端默认目录和 AgentDock 托管配置目录中的本地会话。
- 新增 Antigravity Gemini CLI 会话统计，支持输入、输出、缓存、思考和工具 Token。
- 支持新版 Grok 会话中的上下文 Token 汇总及真实助手请求次数。
- 从客户端页面启动 Grok 时，会把当前选择的项目目录显式传递给 Grok CLI。
- Anthropic 兼容供应商测试能够识别 Claude Code 的 `[1M]` 上下文标记。

### Bug 修复

- 修复通过 AgentDock 使用多次 Codex、Claude Code、Grok 或 Antigravity 后，统计页面仍然没有数据的问题。
- 统计来源现在会去重；Antigravity 重复写入的流式 Token 记录不会被重复计数。
- 修复新版 Grok 不再写入旧 Token 字段后，已有会话无法进入统计的问题。
- 修复 Grok 只打开客户端但没有识别 AgentDock 当前项目的问题；启动器现在同时切换目录并传入官方 `--cwd` 参数。
- 修复火山 AgentPlan 配置使用 `ark-code-latest[1M]` 时，供应商测试返回 `404 UnsupportedModel` 的问题。
- 供应商测试只在实际 API 请求中移除 `[1M]` 后缀，Claude Code 的本地 1M 配置保持不变。

## English

This release consolidates improvements since `v0.1.25`: it fixes empty managed-client usage statistics, Grok project-directory loading, and Volcengine AgentPlan tests returning 404 when the 1M model marker is enabled.

### New Features

- Reads local sessions from both client default directories and AgentDock-managed configuration directories.
- Adds Antigravity Gemini CLI usage statistics for input, output, cached, thinking, and tool tokens.
- Supports context-token summaries and actual assistant request counts from current Grok sessions.
- Explicitly passes the currently selected project directory to Grok CLI when launched from the client page.
- Recognizes Claude Code `[1M]` context markers during Anthropic-compatible provider tests.

### Bug Fixes

- Fixed the statistics page remaining empty after repeated Codex, Claude Code, Grok, or Antigravity use through AgentDock.
- Deduplicates usage sources and repeated Antigravity streaming token records.
- Fixed current Grok sessions being excluded after the client stopped writing legacy token fields.
- Fixed Grok opening without recognizing the current AgentDock project by both changing directories and passing the official `--cwd` option.
- Fixed Volcengine AgentPlan tests returning `404 UnsupportedModel` when configured with `ark-code-latest[1M]`.
- Removes the `[1M]` suffix only from the actual provider test request while preserving the local Claude Code 1M configuration.
