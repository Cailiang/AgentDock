# AgentDock 0.1.32

## 中文

本版本修复从旧版本升级后再次出现 cc-switch 配置导入提示的问题，并恢复稳定的 macOS 应用标识，确保后续升级继续使用已有应用数据。

### 新增功能

- cc-switch 首次检测完成状态现在会持久化到 AgentDock 后端设置，不再只依赖 WebView 本地存储。
- 现有设置文件会自动迁移检测状态，无需用户重新确认或导入已经处理过的 cc-switch 配置。

### Bug 修复

- 修复从 `v0.1.30` 升级到 `v0.1.31` 后重新出现 cc-switch 配置检测和导入页面的问题。
- macOS Bundle ID 恢复为稳定的 `com.agentdock.desktop`，避免不必要的应用标识变化导致 WebKit 数据容器被重置。
- 已导入的 `cc-switch-*` 供应商以及旧版本的本地检测记录会被识别为已完成，避免重复提示。

## English

This release fixes the cc-switch import prompt reappearing after an upgrade and restores the stable macOS application identity so future updates continue using existing app data.

### New Features

- The completed cc-switch initial check is now persisted in AgentDock backend settings instead of relying only on WebView local storage.
- Existing settings files automatically migrate the detection state, so users do not need to reconfirm or re-import previously handled cc-switch configuration.

### Bug Fixes

- Fixed the cc-switch detection and import screen reappearing after upgrading from `v0.1.30` to `v0.1.31`.
- Restored the stable macOS bundle identifier `com.agentdock.desktop`, preventing unnecessary identity changes from resetting the WebKit data container.
- Existing `cc-switch-*` providers and legacy local detection records are recognized as completed to prevent duplicate prompts.
