# AgentDock 0.1.25

## 中文

本版本增加最近项目管理，并修复供应商默认模型保存状态不一致以及 Antigravity 长时间停留在 Thinking 的问题。

### 新增功能

- 客户端的最近项目列表增加独立删除按钮，可以移除已经不再使用的项目记录。
- 删除项目记录只会更新 AgentDock 的最近项目列表，不会删除或修改磁盘中的项目目录。
- 删除当前项目后会同步清空当前选择，并保留项目菜单供用户继续选择其他目录。

### Bug 修复

- 修复编辑 Codex、Antigravity 等非 Claude 供应商时，Claude 模型初始化逻辑错误清空已保存默认模型的问题。
- 修复模型列表异步刷新在供应商编辑器关闭后返回，覆盖下一次编辑中已恢复模型的竞态问题。
- 修复更换默认模型并保存后，再次编辑时列表仍选中第一个模型而不是已保存模型的问题。
- 保存供应商时会把档案中的默认模型同步回 Codex、Grok 和 Antigravity 的高级配置，同时保留其他自定义设置。
- 应用和启动旧供应商配置时会自动修复过期的模型字段，无需用户删除后重新添加供应商。
- Antigravity 启动环境现在强制使用当前供应商的 API Key、Base URL 和 Gemini 模型，避免旧高级配置覆盖新选择后一直停留在 Thinking。

## English

This release adds recent-project management and fixes inconsistent provider default-model persistence and Antigravity sessions remaining in Thinking.

### New Features

- Adds a dedicated remove button to each entry in the client recent-project list.
- Removing a recent project only updates AgentDock history and never deletes or modifies the project directory on disk.
- Removing the current project also clears the current selection while keeping the project menu available for choosing another directory.

### Bug Fixes

- Fixed Claude model initialization clearing the restored default model while editing Codex, Antigravity, and other non-Claude providers.
- Fixed an asynchronous model refresh from a closed provider editor overwriting the restored model in a later editor session.
- Fixed reopening a provider after changing its default model selecting the first available model instead of the saved model.
- Synchronizes the profile default model into Codex, Grok, and Antigravity advanced configuration on save while preserving unrelated custom settings.
- Automatically repairs stale model fields when applying or launching existing provider profiles, without requiring users to recreate them.
- Forces Antigravity launch environments to use the active provider API key, base URL, and Gemini model so stale advanced settings can no longer leave sessions stuck in Thinking.
