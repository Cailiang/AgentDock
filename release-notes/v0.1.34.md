# AgentDock 0.1.34

## 中文

本版本修复 AgentDock 托管客户端与用户配置目录之间的 MCP 检测和同步不一致问题，覆盖 Codex、Claude Code、Grok、OpenClaw 和 Hermes。

### 新增功能

- MCP 导入和同步现在同时覆盖 Codex、Claude Code、Grok、OpenClaw 和 Hermes 的用户配置与 AgentDock 托管运行配置。
- 首次升级会自动导入托管客户端中已有的 MCP，再将统一列表同步回实际运行配置；手动同步会同时更新用户与托管配置。
- OpenClaw MCP 支持 JSON5 配置，并使用当前官方的 `mcp.servers` 结构，同时兼容迁移旧版 `mcpServers` 条目。
- 安装新客户端后会自动补做一次 MCP 同步，已有 MCP 可以立即用于新安装的客户端。

### Bug 修复

- 修复在 AgentDock 新增 MCP 后，启动托管 Codex 看不到新服务器的问题。
- 修复托管 Codex 中原有 MCP 无法通过“导入已有配置”检测的问题。
- 修复 Claude Code 使用托管 `CLAUDE_CONFIG_DIR` 时看不到用户 MCP、托管 MCP 也无法导入的问题。
- 修复 Grok 托管 `GROK_HOME/config.toml` 中已有 MCP 无法导入的问题。
- 修复 OpenClaw 自定义配置路径未收到 MCP 同步，以及旧 `mcpServers` 格式不符合当前官方配置规范的问题。
- 修复 Hermes 使用托管 `HERMES_HOME` 时仍只同步用户 `~/.hermes/config.yaml` 的问题。
- 修复只有 `url`、没有显式 `type` 的 Codex 远程 MCP 被错误识别为 stdio 并跳过的问题。
- 一次性迁移完成后不再重复导入旧的托管配置，避免已删除的 MCP 被重新创建。

## English

This release fixes inconsistent MCP detection and synchronization between AgentDock-managed client homes and user configuration directories for Codex, Claude Code, Grok, OpenClaw, and Hermes.

### New Features

- MCP import and synchronization now cover both user and AgentDock-managed runtime configurations for Codex, Claude Code, Grok, OpenClaw, and Hermes.
- On the first upgrade, existing MCP servers are imported from managed client homes before the unified list is synchronized back to the active runtime configurations; manual synchronization updates both user and managed configurations.
- OpenClaw MCP supports JSON5 and writes the current official `mcp.servers` structure while migrating legacy `mcpServers` entries.
- Installing a new client now triggers MCP synchronization so existing servers are immediately available to that client.

### Bug Fixes

- Fixed newly added AgentDock MCP servers not appearing when managed Codex is launched.
- Fixed existing MCP servers in the managed Codex home not being detected by the import action.
- Fixed Claude Code missing user MCP servers when launched with a managed `CLAUDE_CONFIG_DIR`, and fixed managed Claude MCP servers not being imported.
- Fixed MCP servers in the managed Grok `GROK_HOME/config.toml` not being imported.
- Fixed MCP synchronization missing OpenClaw's custom config path and writing the obsolete `mcpServers` structure instead of the current official schema.
- Fixed Hermes synchronizing only `~/.hermes/config.yaml` while running with a managed `HERMES_HOME`.
- Fixed remote Codex MCP servers with a `url` but no explicit `type` being misclassified as stdio and skipped.
- The one-time migration does not re-import old managed configurations after completion, preventing deleted MCP servers from being recreated.
