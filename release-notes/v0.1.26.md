# AgentDock 0.1.26

## 中文

本版本修复从 AgentDock 启动 Grok 时没有加载当前项目的问题。

### 新增功能

- 从客户端页面启动 Grok 时，会把当前选择的项目目录显式传递给 Grok CLI。
- macOS、Windows 和 Linux 的托管启动器统一支持客户端专用启动参数。

### Bug 修复

- 修复 Grok 只打开客户端但没有识别 AgentDock 当前项目的问题。
- Grok 启动器现在同时切换终端工作目录并传入官方 `--cwd` 参数，避免终端配置或客户端初始化覆盖项目路径。
- 项目路径中的空格和单引号会被安全转义，不会导致启动失败或加载错误目录。

## English

This release fixes Grok not loading the current project when launched from AgentDock.

### New Features

- Explicitly passes the currently selected project directory to Grok CLI when launched from the client page.
- Adds consistent support for client-specific launch arguments across managed macOS, Windows, and Linux launchers.

### Bug Fixes

- Fixed Grok opening without recognizing the current AgentDock project.
- Grok launchers now both change the terminal working directory and pass the official `--cwd` option, preventing terminal profiles or client initialization from overriding the project path.
- Safely quotes project paths containing spaces or single quotes so they cannot break launch or select the wrong directory.
