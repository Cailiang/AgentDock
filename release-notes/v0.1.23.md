# AgentDock 0.1.23

## 中文

本版本修正添加供应商页面的模型列表刷新逻辑，让列表准确反映供应商实际返回的模型。

### 新增功能

- 模型列表会明确区分供应商实际返回和客户端内置推荐两种来源。
- Gemini 模型列表支持分页，并自动排除仅支持 Embedding、不能生成内容的模型。
- 模型接口失败时显示实际请求地址、HTTP 状态和供应商错误信息，便于检查 Base URL、协议和密钥。

### Bug 修复

- 修复供应商模型列表中混入当前供应商并不支持的 AgentDock 推荐模型的问题。
- 修复 Gemini 自定义路径和已包含 `/v1`、`/v1beta` 的地址可能拼接出错误模型接口的问题。
- 修复模型列表请求同时发送多种认证头、与真实客户端认证方式不一致的问题。
- 修复接口不可用时静默显示推荐模型、让用户误以为获取成功的问题。

## English

This release corrects model-list refresh behavior in the provider editor so the list accurately reflects models returned by the provider.

### New Features

- Clearly distinguishes models returned by the provider from built-in client recommendations.
- Supports paginated Gemini model lists and filters out embedding-only models that cannot generate content.
- Shows the attempted endpoint, HTTP status, and provider error when model discovery fails, making Base URL, protocol, and credential issues easier to diagnose.

### Bug Fixes

- Fixed AgentDock recommendations being mixed into the model list even when the provider did not support those models.
- Fixed malformed Gemini model endpoints for custom paths and URLs that already include `/v1` or `/v1beta`.
- Fixed model discovery sending multiple authentication headers that did not match the actual client protocol.
- Fixed unavailable model endpoints silently falling back to recommendations and appearing successful.
