# AgentDock 0.1.31

## 中文

本版本优化 GitHub Actions 发布存储，避免 PR、普通主分支构建和版本发布重复长期保存相同的桌面安装包。

### 新增功能

- 只有 `v*` 版本标签构建才上传跨平台安装包 Artifacts，PR 和普通 `main` 构建仍会完成编译验证。
- 版本标签 Artifacts 的保留时间缩短为 1 天；安装包发布到 GitHub Release 后仍会长期保留。
- Rust 构建缓存只允许从 `main` 分支保存，PR 和标签构建可以读取缓存但不会创建重复副本。

### Bug 修复

- 修复一次发布同时由 `main` 和版本标签构建重复占用约 269 MiB Actions Artifacts 空间的问题。
- macOS 构建不再把未用于 GitHub Release 的 `.app` 目录重复上传到 Artifacts，只保留 universal DMG。
- 修复默认 90 天 Artifact 保留策略导致频繁发布时 Actions 存储快速累积的问题。

## English

This release reduces GitHub Actions storage by preventing pull requests, regular main-branch builds, and version releases from retaining duplicate desktop installers.

### New Features

- Only `v*` tag builds upload cross-platform installer artifacts; pull requests and regular `main` builds still complete compilation checks.
- Tag-build artifacts are retained for one day while installers published to GitHub Releases remain available long term.
- Rust build caches are saved only from `main`; pull request and tag builds may restore caches without creating duplicate copies.

### Bug Fixes

- Fixed each release consuming approximately 269 MiB of duplicate Actions artifact storage from both `main` and tag builds.
- Stops uploading the macOS `.app` directory that is not used by GitHub Releases, retaining only the universal DMG.
- Fixed the default 90-day artifact retention causing Actions storage to grow rapidly during frequent releases.
