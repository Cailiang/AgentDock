# AgentDock 0.1.31

## 中文

本版本为 macOS universal 安装包加入 Developer ID 签名与 Apple 公证，并优化 GitHub Actions 发布存储，减少重复保存桌面安装包。

### 新增功能

- 只有 `v*` 版本标签构建才上传跨平台安装包 Artifacts，PR 和普通 `main` 构建仍会完成编译验证。
- 版本标签 Artifacts 的保留时间缩短为 1 天；安装包发布到 GitHub Release 后仍会长期保留。
- Rust 构建缓存只允许从 `main` 分支保存，PR 和标签构建可以读取缓存但不会创建重复副本。
- macOS 标签构建会在临时钥匙串中导入 Developer ID 证书，签名并公证 `.app` 和最终 universal DMG。
- 发布前会验证 Developer ID 身份、Team ID、Gatekeeper 评估、公证票据、DMG 完整性以及 `arm64`/`x86_64` 双架构。
- macOS Bundle ID 统一为 `com.shuzilm.agentdock`，本地 Mac mini 部署流程同步使用新的应用标识。

### Bug 修复

- 修复一次发布同时由 `main` 和版本标签构建重复占用约 269 MiB Actions Artifacts 空间的问题。
- macOS 构建不再把未用于 GitHub Release 的 `.app` 目录重复上传到 Artifacts，只保留 universal DMG。
- 修复默认 90 天 Artifact 保留策略导致频繁发布时 Actions 存储快速累积的问题。
- 修复公开下载的 macOS 预览包缺少 Developer ID 签名与 Apple 公证而触发 Gatekeeper 安全提示的问题。
- 固定校验并导入 Apple Developer ID G2 中间证书，避免 CI 因证书链不完整而无法识别签名身份。
- 标签构建结束后会移除临时签名钥匙串，避免证书残留在构建环境中。

## English

This release adds Developer ID signing and Apple notarization for the universal macOS installer while reducing duplicate desktop installer storage in GitHub Actions.

### New Features

- Only `v*` tag builds upload cross-platform installer artifacts; pull requests and regular `main` builds still complete compilation checks.
- Tag-build artifacts are retained for one day while installers published to GitHub Releases remain available long term.
- Rust build caches are saved only from `main`; pull request and tag builds may restore caches without creating duplicate copies.
- macOS tag builds import the Developer ID certificate into a temporary keychain, then sign and notarize both the `.app` and final universal DMG.
- Release builds verify the Developer ID identity, Team ID, Gatekeeper assessment, stapled notarization tickets, DMG integrity, and both `arm64` and `x86_64` architectures.
- The macOS bundle identifier is now `com.shuzilm.agentdock`, and the Mac mini deployment flow uses the same application identifier.

### Bug Fixes

- Fixed each release consuming approximately 269 MiB of duplicate Actions artifact storage from both `main` and tag builds.
- Stops uploading the macOS `.app` directory that is not used by GitHub Releases, retaining only the universal DMG.
- Fixed the default 90-day artifact retention causing Actions storage to grow rapidly during frequent releases.
- Fixed downloaded macOS preview packages triggering Gatekeeper warnings because Developer ID signing and Apple notarization were missing.
- Pins, verifies, and imports Apple's Developer ID G2 intermediate certificate so CI can build a complete signing chain.
- Removes the temporary signing keychain after tag builds to avoid leaving certificate material in the runner environment.
