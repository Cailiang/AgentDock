# AgentDock 0.1.20

## 中文

本版本修复 Grok 自定义供应商启动后绕过代理的问题。

### 新增功能

- Grok 启动时会从实际 `config.toml` 读取默认模型别名，支持高级配置中自定义的模型名称。

### Bug 修复

- 修复 AgentDock 无条件注入 `GROK_DEFAULT_MODEL=agentdock`，覆盖用户 `[models].default` 配置的问题。
- 修复自定义配置未定义 `agentdock` 模型时，Grok 回退到内置 xAI 模型并绕过代理地址的问题。
- 清理新保存 Grok 配置中的硬编码默认模型环境变量，密钥仍通过安全的本地存储和启动环境注入。

## English

This release fixes Grok custom providers bypassing their configured proxy after launch.

### New Features

- Grok launches now derive the default model alias from the active `config.toml`, including custom aliases defined in advanced configuration.

### Bug Fixes

- Fixed AgentDock always injecting `GROK_DEFAULT_MODEL=agentdock` and overriding the user's `[models].default` setting.
- Fixed Grok falling back to a built-in xAI model and bypassing the proxy when a custom configuration did not define an `agentdock` model.
- Removed the hard-coded default-model environment value from newly saved Grok settings while keeping API keys in protected local storage and the launch environment.
