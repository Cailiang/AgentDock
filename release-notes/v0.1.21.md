# AgentDock 0.1.21

## 中文

本版本让 AgentDock 安装的命令行客户端可以直接从系统终端启动。

### 新增功能

- 在 `~/.agentdock/bin` 为 AgentDock 托管的 Codex、Claude Code、Antigravity、Grok、OpenCode、OpenClaw 和 Hermes 创建终端命令。
- 自动将 AgentDock 命令目录加入当前用户的 Shell `PATH`；重新打开终端后即可直接执行客户端命令。
- 终端命令通过 AgentDock 的 CLI 代理模式启动，每次动态加载当前供应商、密钥和托管配置，与应用内启动保持一致。
- 升级后会自动为已经安装的客户端补齐终端命令，无需重新安装客户端。

### Bug 修复

- 修复 AgentDock 安装的 Grok 只能从 AgentDock 内启动、直接执行 `grok` 提示命令不存在的问题。
- 修复直接执行托管客户端时可能缺少供应商环境并回退到官方服务的问题。
- 卸载托管客户端时会同步删除 AgentDock 创建的终端命令。

## English

This release makes command-line clients installed by AgentDock directly available from the user's terminal.

### New Features

- Creates terminal commands in `~/.agentdock/bin` for AgentDock-managed Codex, Claude Code, Antigravity, Grok, OpenCode, OpenClaw, and Hermes installations.
- Adds the AgentDock command directory to the current user's shell `PATH`; commands become available after reopening the terminal.
- Routes terminal commands through AgentDock's CLI proxy mode so the active provider, credentials, and managed configuration match in-app launches.
- Repairs command shims for already installed clients automatically after upgrading, without requiring a reinstall.

### Bug Fixes

- Fixed AgentDock-installed Grok only being launchable from AgentDock while `grok` was missing in a regular terminal.
- Fixed direct managed-client launches potentially missing provider environment variables and falling back to official services.
- Removes AgentDock-created terminal commands when uninstalling a managed client.
