# AgentDock 0.1.30

## 中文

本版本合并社区贡献，修复 Windows 启动 AgentDock 桌面应用及执行后台检测命令时出现终端窗口的问题。

### 新增功能

- Windows 桌面版现在使用 GUI 子系统启动，不再为主窗口额外创建控制台窗口。
- 通过 AgentDock 托管命令运行 Codex、Claude Code 等客户端时，会重新连接父终端，继续保留命令输出和退出状态。

### Bug 修复

- 修复 Windows 启动 AgentDock 时短暂弹出黑色终端窗口的问题。
- 修复 Windows 后台检测客户端版本时，PowerShell 或其他子进程窗口闪现的问题。
- Windows 后台命令统一使用无窗口创建标记，避免定时检测和启动准备过程干扰用户操作。

## English

This release merges a community contribution that prevents terminal windows from appearing when AgentDock starts or runs background detection commands on Windows.

### New Features

- The Windows desktop build now starts as a GUI-subsystem application without creating a separate console window for the main UI.
- AgentDock-managed Codex, Claude Code, and other CLI commands reconnect to the parent terminal so command output and exit status remain available.

### Bug Fixes

- Fixed a black terminal window briefly appearing when AgentDock starts on Windows.
- Fixed PowerShell and other child-process windows flashing during background client-version detection on Windows.
- Applies the no-window creation flag consistently to Windows background commands so scheduled detection and launch preparation do not interrupt the user.
