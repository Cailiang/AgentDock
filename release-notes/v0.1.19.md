# AgentDock 0.1.19

## 中文

本版本修复供应商编辑、诊断检测和删除反馈问题。

### 新增功能

- 编辑供应商时会通过独立的本地接口读取已保存的 API Key，并填入密码框；密钥仍不会进入供应商列表、启动缓存或日志。

### Bug 修复

- 修复编辑供应商时已保存的 API Key 不显示的问题。
- 修复诊断中心检测未启用供应商，以及删除供应商后自动启用无关候选配置的问题。
- 升级后会一次性校正旧版本错误启用的 cc-switch 导入记录，之后仍可由用户手动启用，不会被反复覆盖。
- 修复供应商删除成功后列表仍长时间保留旧记录的问题；现在删除成功后会立即更新列表并清除过期诊断结果。

## English

This release fixes provider editing, diagnostic checks, and deletion feedback.

### New Features

- Editing a provider now loads its saved API Key through a dedicated local command and fills the password field. Secrets remain excluded from provider lists, startup caches, and logs.

### Bug Fixes

- Fixed saved API Keys not appearing when editing providers.
- Fixed diagnostics testing inactive providers and provider deletion automatically activating unrelated fallback configurations.
- Added a one-time repair for cc-switch imports incorrectly activated by older versions. Users can still activate those providers manually afterward.
- Fixed deleted providers remaining visible while an unrelated full application refresh completed. The list now updates immediately and stale diagnostic results are cleared.
