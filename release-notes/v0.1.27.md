# AgentDock 0.1.27

## 中文

本版本修复 Grok 项目目录加载，以及火山 AgentPlan 开启 1M 模型标记后测试返回 404 的问题。

### 新增功能

- 从客户端页面启动 Grok 时，会把当前选择的项目目录显式传递给 Grok CLI。
- macOS、Windows 和 Linux 的托管启动器统一支持客户端专用启动参数。
- Anthropic 兼容供应商测试能够识别 Claude Code 的 `[1M]` 上下文标记。

### Bug 修复

- 修复 Grok 只打开客户端但没有识别 AgentDock 当前项目的问题。
- Grok 启动器现在同时切换终端工作目录并传入官方 `--cwd` 参数，路径中的空格和单引号会被安全转义。
- 修复火山 AgentPlan 配置使用 `ark-code-latest[1M]` 时，供应商测试把 `[1M]` 当作模型 ID 发送并返回 `404 UnsupportedModel` 的问题。
- 供应商测试现在只在实际 API 请求中移除 `[1M]` 后缀，Claude Code 的本地 1M 配置保持不变。

## English

This release fixes Grok project-directory loading and Volcengine AgentPlan tests returning 404 when the 1M model marker is enabled.

### New Features

- Explicitly passes the currently selected project directory to Grok CLI when launched from the client page.
- Adds consistent support for client-specific launch arguments across managed macOS, Windows, and Linux launchers.
- Recognizes Claude Code `[1M]` context markers during Anthropic-compatible provider tests.

### Bug Fixes

- Fixed Grok opening without recognizing the current AgentDock project.
- Grok launchers now both change the terminal working directory and pass the official `--cwd` option, with safe quoting for spaces and single quotes.
- Fixed Volcengine AgentPlan tests sending `ark-code-latest[1M]` as the API model ID and receiving `404 UnsupportedModel`.
- Provider tests now remove the `[1M]` suffix only from the actual API request while preserving the local Claude Code 1M configuration.
