# AgentDock 0.1.18

## 中文

本版本完善了 AI 编程客户端的启动流程、工作目录管理、供应商迁移与 Claude Code 模型映射，并修复了 iTerm2 重复窗口、诊断评分失真及供应商编辑和测试等问题。

### 新增功能

- 新增客户端工作目录选择与最近目录记录。启动 Codex、Claude Code 等命令行客户端时，会在所选项目目录中运行。
- 新增 cc-switch 配置迁移，可检测并导入现有供应商配置；导入前自动备份 AgentDock 配置，并对密钥等敏感字段进行安全处理。
- 扩充 Claude Code、Codex、OpenCode、OpenClaw、Hermes 等客户端的内置供应商预设。
- 重做 Claude Code 模型映射配置，支持 fallback、Sonnet、Opus、Fable、Haiku、Subagent、模型显示名称、`[1M]` 上下文选项和认证字段；不同供应商会自动填入对应默认模型，例如火山 AgentPlan 的 `ark-code-latest`。
- 新增客户端安装、升级和启动过程的明确状态反馈；操作期间相关按钮不可重复点击。
- 新增按版本保存的双语 Release Notes，并在中文和英文部分分别列出新增功能与 Bug 修复。
- GitHub 发布脚本和 Actions 工作流现在会在上传安装包前校验 Release Notes 的版本、结构、内容和语言顺序。

### Bug 修复

- 修复从 AgentDock 启动 iTerm2 时可能同时出现一个正确窗口和一个空白窗口的问题，并限制只有明确启动失败时才回退到系统终端。
- 修复客户端启动脚本可能互相覆盖、工作目录没有正确绑定，以及 Grok 安装目录可能被误用于其他客户端的问题。
- 修复静默开机启动和托盘窗口显示行为，避免不需要时显示主窗口或 Dock 图标。
- 修复诊断中心因未安装或未使用的客户端而大幅扣分的问题；评分现在按受影响类别计算，并只检查与已安装客户端相关的供应商和 MCP。
- 修复诊断中心的通过、警告和错误计数无法跳转到对应结果的问题，并移除“导出诊断包”功能。
- 修复火山 AgentPlan 等 Anthropic 兼容供应商测试请求地址和协议不正确而返回 404 的问题。
- 修复编辑从 cc-switch 导入的供应商时，保存按钮标签节点不存在导致 `null is not an object` 的问题。
- 修复 Claude Code 高级选项、模型映射和认证字段未按供应商预设正确保存、读取和应用的问题。
- 修复 GitHub Release 只显示自动生成的 Changelog 链接、没有说明版本实际改动的问题。

## English

This release improves AI coding client launches, working-directory management, provider migration, and Claude Code model mappings. It also fixes duplicate iTerm2 windows, misleading diagnostic scores, and several provider editing and connectivity issues.

### New Features

- Added project working-directory selection and recent-directory history. CLI clients such as Codex and Claude Code now launch in the selected project directory.
- Added cc-switch configuration migration with automatic AgentDock backups and safe handling of secrets during import.
- Expanded built-in provider presets for Claude Code, Codex, OpenCode, OpenClaw, Hermes, and related clients.
- Redesigned Claude Code model mappings with fallback, Sonnet, Opus, Fable, Haiku, Subagent, display-name, `[1M]` context, and authentication-field controls. Provider-specific defaults are filled automatically, such as `ark-code-latest` for Volcengine AgentPlan.
- Added explicit installing, updating, and launching states. Related actions are disabled while an operation is in progress to prevent duplicate requests.
- Added versioned bilingual release notes with separate New Features and Bug Fixes sections in Chinese and English.
- The GitHub release script and Actions workflow now validate the release-note version, structure, content, and language order before uploading installers.

### Bug Fixes

- Fixed iTerm2 opening both the requested client window and an extra empty window. System Terminal fallback now occurs only after an explicit launch failure.
- Fixed launch scripts overwriting one another, incorrect working-directory binding, and the Grok installation path being considered for unrelated clients.
- Fixed silent startup and tray window behavior so the main window and Dock icon stay hidden when appropriate.
- Fixed diagnostics heavily penalizing users for clients they do not install or use. Scoring now penalizes each affected category once and only checks providers and MCP servers relevant to installed clients.
- Fixed the pass, warning, and error counters not navigating to matching diagnostic results, and removed diagnostic bundle export.
- Fixed 404 responses when testing Anthropic-compatible providers such as Volcengine AgentPlan by using the correct Messages endpoint and request format.
- Fixed `null is not an object` when saving a provider imported from cc-switch after the submit-label element was replaced.
- Fixed Claude Code advanced options, model mappings, and authentication fields not being persisted, restored, or applied according to the selected provider preset.
- Fixed GitHub Releases containing only an auto-generated changelog link without explaining the actual changes in the version.
