# AgentDock 0.1.29

## 中文

本版本让 AgentDock 在长期运行时持续发现新版本，并修复 GitHub 公共 API 限流导致更新检查静默失败的问题。

### 新增功能

- 程序启动后立即检查 AgentDock 更新，之后每小时自动检查一次。
- 应用从休眠、隐藏或后台状态回到前台时，如果距离上次检查已超过一小时，会立即补做更新检查。
- GitHub Release 现在随 universal macOS DMG 发布独立 SHA-256 校验文件。

### Bug 修复

- 修复 AgentDock 长时间不重启时无法发现后续新版本的问题。
- 修复同一公网 IP 的 GitHub 未认证 API 额度耗尽后，更新检查返回 `403 rate limit exceeded` 且无法发现新版本的问题。
- 更新查询在 GitHub API 不可用时会回退到 Releases Atom 源，并使用独立校验文件或 GitHub Release 资产摘要继续验证安装包。
- 防止启动、定时器和窗口恢复同时触发重复的更新请求。

## English

This release keeps update discovery active while AgentDock remains open and fixes update checks failing silently when the public GitHub API is rate-limited.

### New Features

- Checks for AgentDock updates immediately after startup and automatically every hour afterward.
- Runs an overdue update check when the app returns from sleep, a hidden state, or the background after more than one hour.
- Publishes a standalone SHA-256 checksum alongside the universal macOS DMG in GitHub Releases.

### Bug Fixes

- Fixed long-running AgentDock instances never discovering versions released after startup.
- Fixed update discovery failing with `403 rate limit exceeded` after the unauthenticated GitHub API quota for a shared public IP was exhausted.
- Falls back to the GitHub Releases Atom feed when the API is unavailable while retaining SHA-256 verification through either the checksum file or GitHub's release-asset digest.
- Prevents startup, timer, and window-resume events from launching duplicate update requests.
