# AgentDock 0.1.24

## 中文

本版本汇总 `v0.1.20` 之后的改进：让 AgentDock 托管的命令行客户端可以直接从终端启动，修复 Antigravity 自定义代理认证，并重做供应商模型发现、选择和真实链路测试。

### 新增功能

- 在 `~/.agentdock/bin` 为 AgentDock 托管的 Codex、Claude Code、Antigravity、Grok、OpenCode、OpenClaw 和 Hermes 创建终端命令，并自动加入用户 Shell `PATH`。
- 终端命令通过 AgentDock CLI 代理动态加载当前供应商、密钥和托管配置；升级后会自动修复已有客户端的命令入口。
- 为托管的 Antigravity Gemini CLI 自动配置 API Key 认证，启动时自动修复旧安装缺失的认证设置。
- 模型刷新会明确区分供应商实际返回与客户端内置推荐；Gemini 列表支持分页，并过滤仅支持 Embedding 的模型。
- 默认模型输入框增加独立的可展开菜单，不再依赖 macOS WebView 支持不稳定的 `datalist`。
- 刷新后直接展示供应商返回的完整模型列表；当前模型不受支持时会明确提示并选择首个可用模型。
- 模型接口失败时显示实际请求地址、HTTP 状态和供应商错误信息。
- Antigravity 供应商测试会使用当前默认模型发送最小生成请求，验证真实对话链路，而不只检查模型列表接口。

### Bug 修复

- 修复 AgentDock 安装的 Grok 只能从应用内启动、直接在终端执行 `grok` 提示命令不存在的问题。
- 修复直接执行托管客户端时可能缺少供应商环境并回退到官方服务的问题；卸载客户端时会同步清理终端命令。
- 修复 Antigravity 自定义供应商测试成功，但启动后仍要求 Google 登录、没有使用代理的问题。
- 修复供应商模型列表混入当前供应商不支持的 AgentDock 推荐模型，以及接口失败时静默显示推荐模型的问题。
- 修复 Gemini 自定义路径和已经包含 `/v1`、`/v1beta` 的地址可能拼接出错误模型接口的问题。
- 修复模型发现同时发送多种认证头、与真实客户端认证方式不一致的问题。
- 修复模型接口显示已返回多个模型，但 macOS 上的默认模型列表看起来没有变化的问题。
- 修复无效默认模型仍被保留，导致 Antigravity 长时间停留在 Thinking 的问题。
- 将托管 Gemini CLI 的聊天请求尝试次数限制为一次，使代理 5xx 错误及时显示。
- 合并 Antigravity 认证和重试配置时保留用户已有的主题、工具及其他 Gemini CLI 设置。

## English

This release consolidates improvements since `v0.1.20`: AgentDock-managed CLI clients are now directly available from the terminal, Antigravity custom-proxy authentication is repaired, and provider model discovery, selection, and end-to-end testing have been rebuilt.

### New Features

- Creates terminal commands in `~/.agentdock/bin` for AgentDock-managed Codex, Claude Code, Antigravity, Grok, OpenCode, OpenClaw, and Hermes clients, and automatically adds the directory to the user's shell `PATH`.
- Routes terminal commands through the AgentDock CLI proxy so they dynamically load the active provider, credentials, and managed configuration; existing installations are repaired automatically after upgrades.
- Automatically configures API-key authentication for the managed Antigravity Gemini CLI and repairs missing authentication settings on launch.
- Clearly separates provider-returned models from built-in client recommendations; Gemini model discovery supports pagination and filters out embedding-only models.
- Adds a dedicated expandable default-model menu instead of relying on inconsistently supported macOS WebView `datalist` behavior.
- Opens the complete provider-returned model list after refresh and clearly selects the first available model when the current model is unsupported.
- Shows the attempted endpoint, HTTP status, and provider error when model discovery fails.
- Tests Antigravity providers with a minimal generation request through the configured default model, validating the real conversation path instead of only the model-list endpoint.

### Bug Fixes

- Fixed AgentDock-installed Grok only being launchable from AgentDock while `grok` was missing in a regular terminal.
- Fixed direct managed-client launches potentially missing provider environment variables and falling back to official services; uninstalling a client now also removes its terminal commands.
- Fixed Antigravity custom providers passing connection tests but still opening the Google login flow and bypassing the proxy at launch.
- Fixed provider model lists being mixed with unsupported AgentDock recommendations and unavailable endpoints silently appearing successful through recommendation fallbacks.
- Fixed malformed Gemini model endpoints for custom paths and URLs that already include `/v1` or `/v1beta`.
- Fixed model discovery sending multiple authentication headers that did not match the real client protocol.
- Fixed model endpoints reporting multiple models while the default-model list appeared unchanged on macOS.
- Fixed unsupported default models remaining selected and leaving Antigravity in Thinking for an extended period.
- Limits managed Gemini CLI chat attempts to one so proxy 5xx errors surface promptly.
- Preserves existing Gemini CLI themes, tools, and other user settings when merging Antigravity authentication and retry configuration.
